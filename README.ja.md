## 💆 自動状態異常回復 とは？

**自動状態異常回復** は、Wear OS スマートウォッチ用の無料のオープンソースアプリで、その場でストレスを管理するのに役立ちます。Fitbit がストレス反応を検出すると（Body Response/EDA センサーを使用）、**自動状態異常回復** は自動的にウォッチ上で触覚振動を使用して、穏やかな 8 秒間のガイド付き呼吸セッションを開始します。

手首にマインドフルネスコーチがいて、ストレスが襲ってきたときにいつでも呼吸とリラックスを助けてくれると考えてください。

### ✨ 気に入っていただける理由

- **🎯 完全自動**：最も必要なときに自動で検出し、呼吸を促します
- **🔋 バッテリー負荷ゼロ**：ストレス検出時のみ動作するため、待機電力ゼロ
- **💆 穏やかなガイダンス**：洗練された振動パターンで呼吸リズムを指引
- **🆓 100% 無料**：オープンソース、広告なし、サブスクリプションなし
- **🔒 プライベート**：完全オフライン - 生体データがデバイス外に出ることはありません

## 📲 インストール方法

### ステップ 1：アプリをダウンロード

1. [**リリースページ**](https://github.com/Ethan-Ming/Auto-Stress-Calm/releases/latest)にアクセス
2. 最新の **`Auto-Stress-Calm-v1.0.1.apk`** ファイルをコンピューターにダウンロード

### ステップ 2：ウォッチにインストール

アプリをインストールするには、ADB（Android Debug Bridge）を使用する必要があります。心配しないでください - 聞こえるよりも簡単です！

#### 🪟 Windows ユーザー向け：

1. **ウォッチで開発者モードを有効にする**：
   - ウォッチで：設定 → システム → バージョン情報 → 「ビルド番号」を 7 回タップ
   - 「あなたは開発者になりました！」と表示されます

2. **ADB デバッグを有効にする**：
   - 設定 → 開発者向けオプション → 「ADB デバッグ」をオンにする
   - 「Wi-Fi 経由でデバッグ」をオンにする
   - 表示される IP アドレスをメモする（例：`192.168.1.100:5555`）

3. **コンピューターに ADB をインストール**：
   - Google から [Platform Tools](https://developer.android.com/tools/releases/platform-tools) をダウンロード
   - zip ファイルをフォルダーに解凍（例：`C:\platform-tools`）

4. **接続してインストール**：
   ```powershell
   # platform-tools フォルダーで PowerShell を開く
   # ウォッチに接続（ウォッチの IP に置き換える）
   .\adb connect 192.168.1.100:5555
   
   # アプリをインストール（APK をダウンロードした場所のパスに置き換える）
   .\adb install Auto-Stress-Calm-v1.0.1.apk
   ```

#### 🍎 Mac/Linux ユーザー向け：

```bash
# ADB をインストール（Mac で Homebrew を使用）
brew install android-platform-tools

# または Linux で
sudo apt-get install adb

# ウォッチに接続
adb connect 192.168.1.100:5555

# アプリをインストール
adb install Auto-Stress-Calm-v1.0.1.apk
```

### ステップ 3：権限を付与

これが最も重要なステップです！アプリには通知を読む権限が必要です。

1. **ウォッチで 自動状態異常回復 を開く**
2. チェックリストが表示されます - **「設定を開く」**をタップ
3. リストで **「自動状態異常回復」**を見つけて**オン**に切り替える
4. プロンプトが表示されたら**「許可」**をタップ
5. アプリに戻る - 緑色のチェックマーク ✅ が表示されるはずです

**これで完了です！** 自動状態異常回復 がストレス通知を監視しています。

## 🎯 使い方

### 自動モード（推奨）

インストールして権限を付与すると、**自動状態異常回復** は自動的に動作します：

1. **Fitbit を着用**して日常生活を送る
2. Fitbit がストレス反応を検出すると、ウォッチに通知を送信します
3. **自動状態異常回復がこの通知をインターセプト**し、すぐに穏やかな呼吸セッションを開始します
4. **振動パターンを感じる**：
   - 🌬️ **息を吸う**（2.5 秒）- 振動が徐々に強くなる
   - 🫁 **保持**（3 秒）- 穏やかな一時停止
   - 😌 **息を吐く**（2.5 秒）- 振動が徐々に弱くなる
5. 一日を通して必要に応じて繰り返す

### 手動テスト

ストレスを待たずに試してみたいですか？

1. ウォッチで **自動状態異常回復** アプリを開く
2. **「触覚をテスト」**ボタンをタップ
3. 8 秒間の呼吸パターンを感じる

## ❓ よくある質問

### 他のフィットネストラッカーでも動作しますか？

現在、Auto Stress Calm は Body Response/EDA ストレス検出機能を備えた **Fitbit** デバイス向けに設計されています。ただし、ストレス通知を送信する他のアプリと連携するように構成できます。

### バッテリーを消費しますか？

いいえ！**自動状態異常回復** は、ストレス通知を受信したときにのみアクティブになるため、バッテリーをほとんど使用しません。効率的に設計されており、バックグラウンドで静かに実行されます。

### データは収集されますか？

**絶対にありません。** Auto Stress Calm は 100% オープンソースで、完全にウォッチ上で実行されます。サーバーにデータが送信されることはなく、インターネット接続も必要ありません。

### 振動を感じない場合はどうすればよいですか？

以下を確認してください：
- ✅ 通知アクセス権限を付与した
- ✅ ウォッチが「おやすみモード」になっていない
- ✅ ウォッチの振動が設定でオンになっている
- ✅ アプリの「触覚をテスト」ボタンを試す

### 呼吸パターンをカスタマイズできますか？

まだできませんが、これは計画されている機能です！現在の 8 秒パターン（2.5 秒吸気、3 秒保持、2.5 秒呼気）は、ストレス緩和のための実証済みの呼吸技術に基づいています。

### どのウォッチがサポートされていますか？

Auto Stress Calm は以下で動作します：
- ✅ Pixel Watch 2（テスト済み）
- ✅ Pixel Watch 3
- ✅ 任意の Wear OS 5 または 6 デバイス（API 31+）

## 🆘 トラブルシューティング

### 「通知アクセスが付与されていません」エラー

1. ウォッチで：**設定 → アプリ → 特別なアプリアクセス → 通知アクセス**
2. **自動状態異常回復** を見つけて**オン**に切り替える
3. **自動状態異常回復** アプリを再起動

### アプリがストレス通知に応答しない

1. **Fitbit アプリ**がウォッチにインストールされ、正常に動作していることを確認
2. Fitbit が通知を送信していることを確認（ストレス反応をトリガーしてテスト）
3. **自動状態異常回復** を開き、緑色のチェックマークが表示されていることを確認
4. ウォッチを再起動してみる

### ADB 経由でインストールできない

- ウォッチで **ADB デバッグ**が有効になっていることを確認
- 接続されていることを確認：`adb devices` でウォッチが表示されるはずです
- 切断して再接続してみる：`adb disconnect` してから `adb connect <IP>`
